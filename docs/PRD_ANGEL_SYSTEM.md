# PRD — 发布后的天使系统（MVP）

> 变更日期：2026-01-25
> 定位：AI在“发布后”作为桥梁与见证者；不做强社交，所有能力基于用户授权与开关。

## 1. 背景与目标

### 1.1 背景
- 发布后内容易沉没，地图与用户缺少轻连接。
- 用户缺少回访与复用动机。

### 1.2 目标
- 让公开/漂流瓶内容形成“可逛的微展”，提升发现与传播。
- 用“回声”提供安全的共鸣连接，不引导长聊。
- 用“时间胶囊”提升回访与留存，让片刻成为长期记忆资产。
- 全程保持用户掌控：默认不打扰、默认关闭敏感能力、随时退出。

### 1.3 成功指标（North Star）
- 发布后7日回访率提升。
- 内容被发现（微展点击/停留/收藏）提升。
- 回声互动提升且投诉率低。
- 时间胶囊触达后二次打开率提升。

## 2. 全局原则（必须遵守）
1. 所有天使行为必须建立在用户授权之上（发布权限 + 天使开关）。
2. 默认克制：不推送轰炸、不强打断、不进入长对话。
3. 所有互动都是轻量卡片：一句话 + 2个按钮。
4. 所有功能可一键关闭/退出，并对单条片刻生效。
5. 私密内容不参与公共微展/回声，仅用于时间胶囊（且可关闭）。

## 3. 权限与开关体系

### 3.1 发布权限（每条片刻）
- 可见性：public / bottle / private。
- 地图展示：允许 / 不允许（默认：public=允许，bottle/private=不展示）。
- 允许被收录微展：开/关（默认关，或按策略）。
- 允许参与回声匹配：开/关（默认关）。
- 允许时间胶囊回访：开/关（默认开，可全局关闭）。

### 3.2 天使模式（每条片刻，发布完成页出现）
- “让天使偶尔路过”：On/Off（默认 Off）。
- 强度：低/中/高（MVP只做低）。
- 频控：每天最多一次天使卡（全局），每条片刻最多一次触发（MVP）。

硬规则：用户未开启天使，则不出现任何天使卡。

## 4. 数据结构（需求级字段）

### 4.1 Moment（片刻）
- moment_id
- user_id
- created_at
- location（可选）
- visibility: public / bottle / private
- media: image / mp4 / cover
- voice_audio（可选）
- transcript_text（可选）
- caption_segments（字幕分段）
- tags（情绪/主题，可选）
- angel_enabled（bool）
- angel_mode（low/med/high）
- allow_microcuration（bool）
- allow_echo（bool）
- allow_timecapsule（bool）

### 4.2 AngelEvent（天使事件）
- event_id
- user_id
- moment_id（可为空）
- type: microcuration / echo / timecapsule
- state: pending / triggered / dismissed / completed
- scheduled_time
- delivered_channel: in-app card / notification（MVP建议只做 in-app）
- cooldown_until

## 5. 模块一：地图微策展

### 5.1 定义
- 基于地图与位置聚合，将公开且允许的片刻按主题/情绪/时间生成“附近微展”。

### 5.2 用户入口
- 地图页新增入口：附近微展（500m / 1km 切换）。
- 天使卡入口：命中触发时出现“附近有一个小展”。

### 5.3 展览形态（MVP）
- 每个微展包含 6–12 条片刻（卡片流）。
- 展名自动生成（短句，运营可兜底模板）。
- 展览筛选：最近24h/7天内；同主题聚合。

### 5.4 策展逻辑（需求层）
- 输入：用户位置/城市、时间窗、公开且允许收录的 moments。
- 输出：若干 micro-exhibits，每个绑定主题/情绪/地点范围。
- 约束：
  - 不展示漂流瓶（除非策略允许）。
  - 不展示私密。
  - 同一用户内容同一微展最多 N 条。
  - 同一用户每天看到推荐微展卡最多 1 次（MVP）。

### 5.5 交互
- 微展内仅轻互动：收藏/点赞/“我也想生成一个同主题片刻”。
- 无私信/对话入口。

### 5.6 用户掌控
- 片刻级开关：不允许被收录。
- 用户级开关：不看微展推荐（仍可手动打开）。

### 5.7 埋点指标
- 曝光、点击、停留时长。
- 收藏/点赞率。
- 从微展到“生成同主题片刻”的转化率。
- 负反馈（关闭、举报、退出收录）。

## 6. 模块二：回声机制

### 6.1 定义
- AI作为桥梁，将两个相似主题/情绪/地点体验的片刻以“共鸣卡”连接。

### 6.2 触发前提（硬条件）
- 两边片刻需满足：
  - visibility=public 或（可选）bottle 且 allow_echo=true。
- 目标用户必须开启天使模式，否则不推回声卡。

### 6.3 回声卡形态（MVP）
- 卡片包含：
  - 你的片刻缩略（封面 + 一句字幕）。
  - TA的片刻缩略（封面 + 一句字幕，匿名）。
  - 一句桥梁文案（模板/AI）。
- 按钮仅两个：
  - “收下这份回声”
  - “不需要/别再推类似回声”

### 6.4 匹配逻辑（需求层）
- 相似度来源：标签/主题、语音文本关键词（需授权）、地点相近、时间相近。
- 约束：
  - 不展示可识别信息。
  - 不提供私聊入口。
  - 每用户每天最多触发1次回声卡（MVP）。
  - 连续2次“不需要”→ 冷却30天。

### 6.5 安全与风控
- 卡片需带“举报/不感兴趣”。
- 屏蔽：可屏蔽某条回声来源。
- 敏感内容：MVP可直接不参与回声池。

### 6.6 埋点指标
- 曝光→点击→收下率。
- 不需要率与冷却触发率。
- 举报/屏蔽率。
- 收下后对回访/再发布提升。

## 7. 模块三：时间胶囊

### 7.1 定义
- 用户自己的片刻在合适时间轻推回，形成回访与自我见证。

### 7.2 开关与默认策略
- 用户级开关：时间胶囊（默认开）。
- 片刻级开关：是否参与回访（默认开）。
- 触达渠道：MVP仅站内卡片（不做系统推送）。

### 7.3 触发策略（MVP）
- 3天后一次。
- 30天后一次。
- 1年后同日一次（后续增强）。

### 7.4 时间胶囊卡（MVP）
- 内容：封面/缩略 + 首段字幕 + 轻提示。
- 按钮：
  - “打开回看”
  - “今天不看/别再提醒这条”

### 7.5 用户掌控
- 可关闭所有时间胶囊。
- 可对单条片刻关闭。
- 可设置静默周期（7天不再提醒）。

### 7.6 埋点指标
- 触发→打开率。
- 打开后生成新片刻/追加语音率。
- 关闭率与静默率。

## 8. MVP范围与不做事项
- 不做强聊天/陪聊。
- 不做复杂社交关系链。
- 微展不做复杂推荐算法，先规则+轻聚合。
- 回声不展示身份、不做对话。
- 时间胶囊不做系统推送。
